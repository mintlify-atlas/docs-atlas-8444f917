Memory Pak is a cross-platform retro game and console collection tracker built with Rust and egui. It lets you mark every console and game you own, have favorited, or want on your wishlist — along with free-text notes on each item. All game data is compiled directly into the binary, so the app works entirely offline with no external dependencies and no account required.

## What you can track

Memory Pak organizes your collection into four dedicated tabs. Each tab has its own search, sort, and filter controls, and every item supports the same Owned / Favorite / Wishlist states.

## Key features

- **Owned / Favorite / Wishlist states** — any item can hold one or more statuses simultaneously, each with a free-text notes field
- **Embedded game database** — all game data is compiled into the binary using `include_dir!()`, so no external files are needed at runtime
- **30+ retro platforms** — covers consoles from Nintendo, Sega, Sony, and Microsoft
- **Cross-console search** — search game titles across every platform at once; queries are Unicode-normalized for consistent results
- **Import / Export** — save your entire collection to a portable JSON file and restore it on any device or platform
- **Responsive UI** — adapts to wide desktop windows and narrow mobile screens; screens under 700 px wide get a hamburger menu (☰) to toggle the sidebar
- **Web (WASM) support** — runs in the browser at [aspenini.github.io/Memory-Pak](https://aspenini.github.io/Memory-Pak) with state stored in browser localStorage

## Platform support

Memory Pak runs on all major desktop platforms and in the browser. Your data is stored in the standard user data directory for your OS, or in localStorage on the web.

| Platform | Data location |
|----------|--------------|
| Windows  | `%APPDATA%\com\memorypak\memory_pak\state\` |
| macOS    | `~/Library/Application Support/com.memorypak.memory_pak/state/` |
| Linux    | `~/.local/share/com/memorypak/memory_pak/state/` |
| Web      | Browser localStorage |
